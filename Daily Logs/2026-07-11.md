
## Work Completed

### Pre-Installation Preparation

Before wiping the M720q's SSD, completed a full pre-flight checklist to avoid losing the existing Windows 11 Pro OEM license and to confirm hardware readiness.

**Windows License Preservation**

- Linked OEM Windows digital license to Microsoft account (Settings → Update & Security → Activation confirmed: "Windows is activated with a digital license linked to your Microsoft account")
- Retrieved and saved OEM product key as a manual fallback using PowerShell: `(Get-CimInstance -ClassName SoftwareLicensingService).OA3xOriginalProductKey`
- Noted `wmic` is deprecated on newer Windows builds (10.0.26200.x) — PowerShell is now the correct tool for this

**BIOS Configuration**

- Confirmed Intel Virtualization Technology (VT-x): Enabled
- Confirmed VT-d: Enabled
- Disabled Secure Boot
- Confirmed UEFI-only boot mode

**Network Planning**

- Identified home subnet via `ipconfig`: `192.168.86.0/24`, gateway `192.168.86.1`
- Verified `192.168.86.200` was unused via `arp -a` before assigning
- Chose static IP over DHCP reservation (no admin access to the Google Wifi app)

**Storage/Filesystem Decision**

- Evaluated ext4+LVM vs ZFS for a single-disk system
- Decided on ext4+LVM — ZFS's main benefits (redundancy, self-healing) require multiple disks to be useful, and its RAM/write overhead isn't worth it on a single consumer SSD
