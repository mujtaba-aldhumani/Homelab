### Ubuntu Server VM + Tailscale

Created a second VM specifically to run Tailscale, for two purposes: general remote access to home infrastructure from anywhere, and acting as a shared proxy so a Stremio + Real-Debrid setup can be used by multiple people without each appearing from a different IP.

* Clarified the actual requirement before building: only Stremio/RD traffic should route through the shared connection — friends' other traffic should stay on their own connections. This rules out Tailscale's default exit-node mode (which is all-or-nothing per device) and instead calls for a dedicated proxy service (e.g. SOCKS5) running on the VM, with Tailscale providing the private network to reach it — friends would point Stremio's own proxy settings at the VM specifically, not enable a system-wide exit node
* Noted a limitation to be aware of: even with all traffic funneled through one IP, Real-Debrid may also track concurrent stream count per account independent of IP, so single-IP routing may not fully eliminate multi-user restrictions
* VM specs: SeaBIOS, i440fx machine type, no TPM (not needed for Linux), VirtIO SCSI disk (20GB, single ext4 partition + LVM, left at 10GB allocated with free space unused for now), VirtIO NIC, 2048MB RAM, 1 core
* Installed Ubuntu Server 26.04 LTS, static IP 192.168.86.201, hostname `tailscaleproxy`, OpenSSH server installed during setup
* Installed Tailscale on the VM and authenticated it to the tailnet (Tailscale IP 100.74.175.81, machine name `tailscaleproxy`)
* Installed Tailscale on the personal laptop (`mujlaptop`) and joined the same tailnet (Tailscale IP 100.115.203.84) — correction: this was the personal laptop, NOT the Windows 11 practice VM, which had not yet been joined to the tailnet at this point
* Verified connectivity end-to-end via ping between the two Tailscale IPs — confirmed working
